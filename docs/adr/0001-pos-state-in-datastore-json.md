# ADR-0001：以 DataStore Preferences＋JSON 束集中承載 POS 狀態

**Date**：2026-05-14  
**Status**：Accepted

## Context

本專案目標客群為市集現場：**裝置離線可操作、架構維護成本要低、介面迭代要快**。需要在單機上保存目錄、購物車、銷貨紀錄與統計數字，並在結帳時盡可能在一次寫入內達成相容性。

若採典型關聯式機制（複雜遷移、多表一致性）會推高維護量，與「減法設計」相悖；但也不能完全零散任意寫檔而失去交易感。

## Decision

將主要持久化收口為：**AndroidX DataStore Preferences**，以數個字串鍵承載對應的 JSON 資料束（含目錄、購物車、銷貨紀錄等），並在結帳／復原等路徑以 `edit { }` **單一區塊**更新相關欄位。

## Alternatives considered

- **SQLite／Room／多資料表模型** — 模型表達力强、查詢彈性高；對本專案規模需在遷移、DAO 與交易邏輯上分攤更多成本，對「單機小攤結帳」而言槓桿不足。
- **多檔案或自管磁碟版面** — 容易出現備份截面與 app 資料夾相容性問題，且缺少 DataStore 的慣例與協程整合。

## Consequences

- **較簡**：多數資料以 JSON 視角讀寫，與市集目錄匯出的思維一致。
- **交易語意**：由應用在單次 `edit` 內自守「結帳＋銷紀＋累計」等動作的一致性；並以文件與測試對帳約束細節。
- **查詢面**：若以報表級 SQL 視角評估會受限；若以單機匯出與有限列表為主則尚可。
